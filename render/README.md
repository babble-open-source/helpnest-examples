# HelpNest — Render Deploy

Deploy HelpNest to Render using the Blueprint. Render provisions the app, PostgreSQL, and Redis automatically from `render.yaml`.

## One-click deploy

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/babble-open-source/helpnest-examples)

## Manual deploy

### 1. Fork the repo

Fork [helpnest/helpnest](https://github.com/babble-open-source/helpnest) to your GitHub account.

### 2. Create a Blueprint

[render.com](https://render.com) → New → **Blueprint** → connect your fork → Render reads `render.yaml` and creates all services.

### 3. Set secret variables

After the blueprint runs, in your Render web service → **Environment**:

- `NEXTAUTH_URL` — your Render URL e.g. `https://helpnest.onrender.com`
- `NEXT_PUBLIC_APP_URL` — same

`NEXTAUTH_SECRET` is auto-generated by the blueprint.

### 4. Run migrations

Render dashboard → your web service → **Shell**:

```bash
npx prisma migrate deploy
```

### 5. Custom domain

Render → Settings → Custom Domains → add your domain → update DNS.

## Enabling the AI Agent

After your first login:

1. Go to **Dashboard → Settings → AI Agent**
2. Toggle **Enable AI Agent** on
3. Choose your **AI Provider** (Anthropic, OpenAI, Google, or Mistral)
4. Enter the **Model** name (e.g. `claude-sonnet-4-5`, `gpt-4o`, `gemini-2.0-flash`)
5. Paste your **API Key** for that provider
6. Click **Save**

For vector search, add to your Render environment variables:

```env
OPENAI_API_KEY=sk-...
QDRANT_URL=https://...   # use Qdrant Cloud free tier
```

## Services created

| Service | Type | Render plan |
|---------|------|-------------|
| `helpnest-app` | Web Service (Docker) | Starter ~$7/mo |
| `helpnest-db` | PostgreSQL | Starter ~$7/mo |
| `helpnest-redis` | Redis | Starter ~$3/mo |

Free tier is available for individual services (with spin-down on inactivity). Use Starter plans for always-on production.
